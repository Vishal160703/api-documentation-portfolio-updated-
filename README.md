# API Documentation Portfolio

## Project 1: User Management API

This project demonstrates API documentation for user-related operations including retrieving and creating users.

### Features:

* GET Users API
* POST Create User API
* Authentication
* Error Handling

---

Authentication:
This API uses Bearer Token authentication. To access the API, include an Authorization header in all requests.
Header Format:
Authorization: Bearer <your_access_token>
Description:
The access token is used to authenticate API requests and ensure secure communication between the client and server. Requests without a valid token may result in a 401 Unauthorized response.


GET

API Name: Get Users
Endpoint: https://jsonplaceholder.typicode.com/users
Method: GET
Description:
This API retrieves a list of user details from the server. It returns user information in JSON format and is commonly used to display user data in applications.
Request:
No request body is required for this API.
Parameters:
•	Query Parameter (id):
Used to filter users based on their ID. For example, ?id=1 returns only the user with ID 1.
•	Path Parameter (user ID):
Used to retrieve a specific user by including the user ID in the URL. For example, /users/1 returns details of the user with ID 1.
Headers:
•	Content-Type: application/json
Specifies that the data format of the request/response is JSON.
•	Authorization: Bearer <token>
Used to authenticate the request by passing an access token.
Response:
The API returns user details without modifying any data. The response includes the following fields:
•	id: A unique identifier assigned to the user.
•	name: The full name of the user.
•	email: The email address associated with the user account.
•	address.city: The city where the user is located.
Example Response:
[
  {
    "id": 1,
    "name": "Leanne Graham",
    "email": "leanne@test.com"
  }
]


POST
API Name: Create User
Endpoint: https://jsonplaceholder.typicode.com/users
Method: POST
Description:
This API creates a new user on the server. It accepts user details in the request body and returns the created user information in JSON format. This API is commonly used in applications for user registration or data creation.
Request Body:
The request body should be sent in JSON format and includes the following fields:
•	name: The full name of the user.
•	email: The email address associated with the user.
•	city: The city where the user is located.
Note: The id is generated by the server and should not be included in the request.
Headers:
•	Content-Type: application/json
•	Authorization: Bearer <token>
Response:
The API returns the created user details along with a generated unique ID.
•	id: A unique identifier generated for the new user.
•	name: The full name of the user.
•	email: The email address associated with the user.
•	city: The city where the user is located.
Example Request:
{
  "name": "Vishal",
  "email": "vishal@test.com",
  "city": "Varanasi"
}
Example Response:
{
  "name": "Vishal",
  "email": "vishal@test.com",
  "city": "Varanasi",
  "id": 11
}

Error Handling Section

Error Responses:
The API returns appropriate HTTP status codes and error messages when a request fails. Common error responses are described below:
•	400 Bad Request
Indicates that the request is invalid or missing required fields.
Example: Missing required field such as email or invalid input format.
•	401 Unauthorized
Indicates that the request lacks valid authentication credentials.
Example: Missing or invalid authentication token.
•	404 Not Found
Indicates that the requested endpoint does not exist.
Example: Incorrect API URL.
•	500 Internal Server Error
Indicates a server-side issue while processing the request.
Example: Unexpected server failure.
Example Error Response:
{
  "error": "Invalid email format"
}
